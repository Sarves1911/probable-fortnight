# LeNet-5 C++ Inference Engine

Zero-dependency C++ inference engine for LeNet-5 with INT8 weight quantization. No PyTorch, no ONNX, no ML framework вЂ” just C++ and the weights.

Runs 10,000 MNIST test images at **9,795 images/sec** (0.102ms/image) on Apple M-series with **98.78% accuracy**.

---

## Motivation

Most inference engines are either Python wrappers around C++ (PyTorch, TensorFlow) or massive frameworks with thousands of lines of abstraction (ONNX Runtime, TensorRT). This project strips all of that away: raw C++ forward pass, INT8 quantized weights loaded from binary files, and a hand-rolled Conv2D, MaxPool2D, and FC stack.

Built as a systems exercise in understanding what happens below the framework layer вЂ” memory layout, quantization arithmetic, and how much I/O overhead costs you (spoiler: gating 10K `std::cout` calls behind `--verbose` gave a 14.5% throughput jump).

## Quantization scheme

Weights are quantized to INT8 using symmetric per-layer quantization at export time (Python/PyTorch). The scale factor and zero point are stored alongside the binary weights in a JSON metadata file.

At inference time, weights are stored as `int8_t` and dequantized per layer before the MAC operations:

```
real_weight = (q_weight - zero_point) * scale
acc += input_activation * real_weight
```

Activations remain float32 throughout. This is weight-only quantization вЂ” the same approach used by GGUF/llama.cpp for LLM inference. The memory footprint of the weight tensors is reduced by ~4Г—; compute is float32 MACs.

| Accuracy             | 98.78% (9878/10000) |
| Total inference time | 1020.84 ms          |



### 3. Run inference

```bash

./build/inference

# Per-sample predictions
./build/inference --verbose
```

## Weight format

Each layer's weights are stored as two files:

`conv1.weight.bin` вЂ” raw `int8_t` values, row-major, shape `[out_ch, in_ch, kH, kW]`

`conv1.weight.json` вЂ” metadata:

```json
{
  "shape": [6, 1, 5, 5],
  "scale": 0.003921568859368563,
  "zero_point": 0
}
```

Biases are stored as raw `float32` with a matching JSON shape file.

## Implementation notes

**Conv2D memory layout** вЂ” NCHW throughout (`batch Г— channels Г— height Г— width`). Flat index computed as `c * H * W + h * W + w`. The 6-deep loop order is `[oc, oh, ow, ic, kh, kw]` which is cache-friendly for the output write.

**Softmax numerical stability** вЂ” subtracts `max(logits)` before `exp()` to prevent float32 overflow. Standard trick, included because the naive version breaks on large logit ranges.

**Error handling** вЂ” every `fread` checks `gcount()` against expected bytes, every shape mismatch throws with a descriptive message. No silent failures.

**I/O note** вЂ” per-sample `std::cout` inside a 10K-image loop costs ~15% throughput. Gated behind `--verbose`. Benchmark numbers are I/O-free.
